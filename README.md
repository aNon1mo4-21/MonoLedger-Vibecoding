# MonoLedger 💰

> **Vibecoding with Gemini Canvas** 🚀

**MonoLedger** 是一个优雅、纯粹、隐私优先的个人资产管理工具。它是一个典型的 **Vibecoding** 产物——由人类提供灵感与“氛围”，由 **Gemini Canvas** 实时编码生成。

它证明了：通过正确的 AI 交互，你不需要从零编写代码，也能拥有一个功能完整、UI 现代且完全属于自己的生产力工具。

## ✨ 核心功能

- **📊 资产大盘**：基于 **Recharts** 的环形图与柱状图，直观展现收支比例与账户分布。
- **💸 智能账目**：
  - 支持支出、收入、以及账户间互转。
  - **小收益模式**：专为懒人设计，只需输入当前余额，系统自动计算并记录利息/收益。
- **☁️ GitHub 云同步**：彻底摆脱后端服务器。利用 GitHub API 将私有仓库作为数据库，实现多端同步与版本备份。
- **🏦 灵活账户管理**：支持自定义账户（支付宝、微信、银行卡等）的初始化与动态管理。
- **📥 数据出口**：支持一键导出 CSV 报表，数据所有权永远在你手中。
- **📱 移动优先**：响应式设计，完美适配手机浏览器，开启 GitHub Pages 后即刻化身 Web App。

## 🚀 如何使用

MonoLedger 采用 **单文件 (Single-File)** 架构，即开即用：

1. **下载**：获取 `V1.0.1.html` 文件。
2. **运行**：双击文件，用任意浏览器打开。
3. **部署（可选）**：将文件上传至 GitHub 仓库并开启 **GitHub Pages**，即可通过自定义域名随时随地记账。

# ☁️ 云同步配置说明

为了安全地通过 GitHub 私有仓库同步你的账单数据，请按以下步骤进行配置。

## 🗂️ 第一步：准备 GitHub 仓库

1.在 GitHub 上创建一个 **Private（私有）** 仓库（例如命名为 `my-ledger-data`）。

2. 在该仓库中手动创建一个名为 **`data.json`** 的空文件，初始内容填写为 `{}`即可。

3. 前往 GitHub Personal Access Tokens生成一个 **具有 `repo`权限** 的 Token。

------

## ⚙️ 第二步：配置同步参数

### 方案 A：通过 UI 界面配置（推荐）

1. 打开应用的 HTML 文件。

2.  点击底部导航栏的 **☁️ 云朵图标（数据同步）**。

3. 点击 **同步参数配置**，填入你的 **GitHub 用户名、仓库名和 Token**。

4. 配置信息将自动保存在浏览器本地存储中，下次打开无需重复输入。

### 方案 B：直接修改源码（适合固定设备）

如果你希望在所有设备打开时自动使用同一配置，可以用文本编辑器打开 HTML 文件，找到第 52 行附近的 `DEFAULT_GIT_CONFIG`常量，填入你的信息：

```
const DEFAULT_GIT_CONFIG = {
    user: "你的用户名",
    repo: "你的仓库名",
    path: "data.json",
    token: "你的 ghp_ 开头的 Token"
};
```

------

## 🔄 第三步：初始化与同步操作

配置完成后，在 **数据同步** 页面进行操作：

- **下载并覆盖本地**：首次使用时，点击此按钮从云端仓库获取 `data.json`并覆盖本地数据。如果成功，会提示“已从云端下载并替换本地数据”。

- **覆盖写入**：当本地账单有新增或修改后，点击此按钮将本地数据推送并覆盖云端文件。

> 提示：首次同步请务必先执行 **下载并覆盖本地** 完成初始化。
